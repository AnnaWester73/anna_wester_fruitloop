Generellt från grundstrukturen i projektet har jag ändrat om i strukturen
för att få en renare arkitektur.

grid.py ansvarar Spelplan.
player.py ansvarar för spelarens data.
pickups.py ansvarar för Items.
game.py ansvarar Spelregler
config.py ansvarar konfiguration av värden
gui.py ansvarar för utskrift och användargränssnitt

Utöver övningsuppgifter har det tillkommit logik för Game over.
För att lättare testa spelet se i config för att ändra start värden:

Följande är löst i projektet
Version 1 Grundkrav:
A.) Start nära mitten
→ player_start_x och player_start_y beräknas från gridens mitt i config.py.

B.) Rörelse i 4 riktningar
→ Konfigurerat via movements i config.py.

C.) Kan inte gå genom väggar
→ Implementerat i Player.can_move().

D.) Alla frukter värda 20 poäng
→ default_item_value = 20 i config.

E. )Inventory
→ Player.inventory + add_item().

F.) Kommando "i"
→ Hanteras i Game.handle_command().

G.) The floor is lava -1 poäng per steg
→ apply_step_penalty().

H.) Sammanhängande väggar via for-loop
→ make_internal_walls() i grid.

Version 2
I.) Fällor - introducera valfri fälla till spelplanen.
→ trap med is_trap=True.
→ Ger -10 poäng.
→ Ligger kvar om man inte desarmerar.

J.) Spade - som adderas och tar bort intern vägg
→ breaks_wall=True.
→ remove_connected_wall() tar bort hela interna väggen.


Version 3
O.) Grace period - efter man har tagit plockat upp något, 5 steg
→ grace_counter i Player.

Q.) Tryck "B" för att placera en bomb.
→ place_bomb() + update_bombs()
→ 3-ticks timer
→ 3x3 explosion
→ Skadar spelare

R.) Ett nytt kommando ("T" för trap) för att desarmera fällor.
→ Kommando "t"
→ disarm_trap()
